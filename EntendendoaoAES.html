O que é a criptografia AES

    Entrada de Dados: O algoritmo AES recebe como entrada: Uma chave de criptografia (128, 192 ou 256 bits). Um bloco de dados a ser criptografado (normalmente 128 bits de tamanho). O bloco de dados é dividido em uma matriz retangular de 4x4 bytes (uma matriz de estado).

    Funciona da seguite forma


    1°- A chave de criptografia é expandida e preparada para ser usada durante várias rodadas de criptografia.
    2°- Cada chave de rodada é derivada da chave original usando um algoritmo específico.
    3°- Rounds (Rodadas): O AES opera em várias rodadas (10, 12 ou 14 rodadas dependendo do tamanho da chave). Cada rodada executa uma série de transformações no bloco de dados.
    SubBytes: Cada byte do bloco de dados é substituído por um valor correspondente em uma S-Box (caixa de substituição) predefinida. Isso fornece confusão nos dados, dificultando a identificação de padrões.
    5°- Agora calculamos D = J⁽⁻¹⁾ mod G (este é o calculo do inverso multiplicativo modular)
    6°- Seja M a string numerica que queremos codificar (Uma mensagem convestida para tabela ASCII)
    MixColumns: Cada coluna da matriz de estado é tratada como um polinômio, e é multiplicada por uma matriz fixa.
    Isso embaralha ainda mais os bytes dentro de cada coluna.
    AddRoundKey: A chave de rodada atual é combinada com o bloco de dados usando uma operação de XOR (OU exclusivo). Isso adiciona uma camada de segurança, misturando os dados com a chave.
    Rodada Final: A rodada final não inclui a etapa MixColumns.
    Resultado: Após a conclusão das rodadas, o bloco de dados é criptografado.
    Decifragem: A decifragem ocorre aplicando o inverso de cada passo na ordem inversa.
    O AES é amplamente considerado seguro e eficiente para criptografia de dados.
    Sua força vem da combinação de várias etapas de transformação, dificultando a recuperação dos dados originais sem a chave de criptografia correta.

    Vamos aplicar em um exemplo

    1. Preparação da Mensagem e da Chave: ◦ Inicialmente, temos a mensagem que queremos criptografar, que é "miojo".
    ◦ Também temos uma chave de criptografia, que é uma sequência de caracteres.
    Neste exemplo, usaremos "chave_secreta" como a chave.
    2. Criptografando a Mensagem:
    ◦ A mensagem "miojo" é representada em formato binário, que é a forma como os computadores lidam com os dados.
    ◦ A chave também é convertida em um formato que o algoritmo AES possa usar.
    ◦ O algoritmo AES então executa várias iterações, ou "rodadas", para embaralhar os dados da mensagem usando operações matemáticas e substituições de bytes.
    ◦ Cada rodada aplica uma série de transformações nos dados da mensagem, misturando-os e obscurecendo-os.
    ◦ Durante essas rodadas, os dados da mensagem são combinados com a chave de criptografia usando operações como XOR (OU exclusivo).
    ◦ O resultado dessas iterações é a mensagem criptografada, que é uma sequência de bytes que não se assemelha mais à mensagem original "miojo".
    3. Descriptografando a Mensagem:
    ◦ Para descriptografar a mensagem, é necessário usar a mesma chave que foi usada para criptografá-la.
    ◦ A mensagem criptografada é passada pelo algoritmo AES junto com a chave de criptografia.
    ◦ O algoritmo AES executa o processo inverso do que fez durante a criptografia, desfazendo as transformações aplicadas à mensagem.
    ◦ Após as iterações necessárias, a mensagem original "miojo" é recuperada.
    Em resumo, o algoritmo AES funciona transformando a mensagem original em uma forma embaralhada e obscurescida, que é praticamente impossível de entender sem a chave correta.
    Essa criptografia fornece uma camada de segurança, garantindo que apenas aqueles com a chave correta possam acessar e entender a mensagem original.
